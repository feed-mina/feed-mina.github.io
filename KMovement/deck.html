00/Why
Motivation

기능을 추가한 게 아니라,
경험을 설명 가능한 흐름으로 바꿨다.

추천 → 설명 → 생성 → 검증. 이 한 줄이 이번 발표의 목차다.

Problem
경로와 단순 추천에서 멈춰 있었다
사용자는 “왜 이 장소가 나에게?”의 답을 받지 못했고, 결과 화면은 이유 없는 리스트였다.
Need
추천에 근거를 붙여야 한다
K-콘텐츠·아티스트·취향이 얽힌 다차원 질의를 풀려면 데이터 구조와 검색 계층을 다시 짜야 했다.
Shift
생성형 AI는 화면 위의 호출이 아니다
5~15분 걸리는 영상 생성을 다루려면 비동기 운영 구조와 데이터 파이프라인이 함께 와야 했다.

02 Architecture

화면에서 모델을 바로 부르지 않는다 — 책임을 4계층으로 분리했다.

TIER 01
Frontend
Next.js · React · SDUI metadata · Community CRUD · Playwright 검증.
›
TIER 02
Backend API
Spring Boot · auth · content · DB transaction. 짧고 잦은 요청을 받는다.
›
TIER 03
AI Orchestrator
FastAPI · GraphRAG · BLIP-2 · TTS · media job 접수. Celery로 비동기 큐 분리.
›
TIER 04
Model Ops
TorchServe · MLflow · Docker · model store. 모델 파일은 API에서 떼어 둔다.
왜 4계층?
짧은 요청과 5~15분 GPU 작업을 같은 방식으로 처리하지 않는다
동기 API가 무거운 추론을 받으면 timeout과 서버 마비로 이어진다. job_id 즉시 발급 → Celery 워커가 뒤에서 처리하는 구조.
왜 TorchServe?
API 서버가 모델 파일을 직접 들고 있지 않다
handler · MAR · model store · endpoint로 운영 단위를 떼어 두면, AI 코드 변경이 API 재배포로 이어지지 않는다.

Inputs
① video.mp4 · ② voice.wav (TTS) · ③ bgm.wav (MusicGen) · ④ subtitle.srt
Filter
-filter_complex "[2:a]volume=-18dB[bgm_low]" — BGM ducking

05 Troubleshooting · 01

Trouble · TTS

언어 코드와 실행 환경 차이에서 시작된, Segmentation Fault까지 간 문제.

Before
WebUI · API · OS 레벨이 따로 놀았다
  • WebUI는 “Korean / 한국어”, API는 “ko”를 요구
  • Mecab C++ 엔진과 한국어 사전(mecab-ko-dic) 경로가 꼬임
  • BigVGAN 로딩 시 huggingface_hub 파라미터 누락
  • 결과: 워커가 에러 로그 없이 즉사 (Segmentation Fault)
→
After
API 기준 언어 코드 + 런타임 몽키 패치
  • API 입력 언어 코드를 ko로 통일, fallback gTTS 경로 추가
  • Mecab 사전 경로(/usr/local/lib/.../mecab_ko_dic/dicdir)를 _g2p.mecab 객체에 직접 매핑
  • BigVGAN._from_pretrained를 실행 시점에 가로채 파라미터 강제 주입 (Monkey Patching)
  • 모델 파일·의존성 체크리스트 문서화 (보고서 §4.2)

05 Troubleshooting · 02

Trouble · GPU & Memory

멋있는 모델보다 운영 구조가 중요했다.

Before
동기 API + 거대 모델이 충돌했다
  • CogVideoX-5b는 15GB VRAM(Kaggle T4)에서 즉시 OOM
  • 의존성 지옥: torchvision · transformers · diffusers · mmcv-full 버전 충돌
  • 한 요청이 5~15분 — 동기 API timeout
  • TorchServe 워커가 에러 로그 없이 즉사 (np.float deprecated)
→
After
샌드박스 격리 + 비동기 큐 + CPU offload
  • 모델별 Miniconda 가상환경 분리 (TorchServe 0.6.0 · Numpy 1.23.5 핀)
  • CogVideoX: enable_sequential_cpu_offload + VAE slicing + tiling
  • Celery + Redis로 작업 분리, FastAPI는 즉시 job_id 반환
  • 3중 생존 방어막: 정상 렌더 → 프레임 추출 후 FFmpeg 강제 조립 → 5초 더미 영상

Trouble · Data & Screen
