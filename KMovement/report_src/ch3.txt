제안해 드린 목차와 방향성에 맞추어, K-Ride 시스템의 가장 핵심적인 인공지능 영상/음성
처리 기술을 다루는 **제3장. 멀티모달 AI 미디어 자동 생성 모델 및 파이프라인**을 작성해
드립니다.
제3장. 멀티모달 AI 미디어 자동 생성 모델 및 파이프라인
본 장에서는 텍스트 형태의 여행 일정과 사용자의 시각적 입력물(사진, 낙서)을 입력받아,
영상, 음성, 음악이 결합된 하나의 완벽한 미디어 콘텐츠(Vlog, 애니메이션)로 자동
변환하는 멀티모달 파이프라인의 핵심 모델 구조와 렌더링 원리를 분석한다. 무거운 이기종
딥러닝 모델들을 제한된 GPU 환경(Kaggle T4 15GB 등)에서 메모리 초과(OOM) 없이
안정적으로 구동하기 위한 지능형 라우팅 및 헤드리스(Headless) 파이프라인 설계에
초점을 맞춘다.
3.1. 시각 정보 기반 지능형 라우팅 로직
단일 고성능 비디오 생성 모델로 모든 이미지를 처리할 경우 막대한 연산 비용과 메모리
부족(OOM) 현상이 발생한다1. 이를 해결하기 위해 K-Ride 시스템은 이미지의 맥락을
분석하여 가장 효율적인 영상 생성 모델로 작업을 분기(Routing)하는 지능형 파이프라인을
도입하였다2.
● BLIP-2 모델을 활용한 이미지 영문 캡셔닝 체계사용자가 업로드한 여행 사진(예:
해변의 풍경, 반려동물과의 사진)은 가장 먼저 시각-언어 모델인
BLIP-2(Bootstrapping Language-Image Pre-training)로 전달된다3. BLIP-2는 입력된
이미지를 분석하여 "A woman sitting on the beach with a dog" 또는 "A wide shot of
beautiful blue ocean waves"와 같은 고품질의 영문 캡션(설명)을 자동으로
추출한다3. 이 캡션 데이터는 이어지는 비디오 생성 모델의 입력
프롬프트(Prompt)로 활용될 뿐만 아니라, 라우팅의 핵심 기준점이 된다2, 3.
● 자원 최적화를 위한 조건부 분기: 동적 모델(CogVideoX-5b) vs 정적 모델(3D Photo
Inpainting)추출된 영문 캡션은 파이썬 기반의 라우터 함수(route_video_model)를
거친다2. 시스템은 캡션 내에 'person', 'people', 'dog', 'animal' 등 동적인 피사체를
의미하는 특정 키워드가 하나라도 포함되어 있는지 검사한다2.동적 키워드가
감지되면, 사람이나 동물의 자연스러운 모션 생성에 탁월하지만 연산량이 무거운
'CogVideoX 1.5 (CogVideoX-5b-I2V)' 모델로 작업을 배정한다2. 반면, 동적
피사체가 없는 단순 자연 풍경이나 건축물 사진의 경우, 5초 내외의 가벼운
패닝(Panning) 효과만으로도 충분한 시각적 만족감을 줄 수 있으므로 연산량이 적은
'3D Photo Inpainting' 모델로 자동 배정한다2. 이와 같은 하이브리드 영상 생성
라우팅 로직은 서버 파이프라인의 처리 소요 시간과 GPU 비용을 획기적으로
감축시킨다4.
3.2. 2D 캐릭터 애니메이션 자동화 (Animated Drawings)
사용자가 사진 대신 러프한 스케치나 낙서(Doodle)를 입력할 경우, Meta의 오픈소스
프레임워크인 'Animated Drawings' 기반의 파이프라인이 가동된다5.
● 포즈 추정(Pose Estimator) 및 뼈대 추출(Detector) 모델 구조낙서 이미지에 생명을
불어넣기 위해 시스템은 컴퓨터 비전 라이브러리인 mmcv-full의 CUDA 연산을
활용한다6. 우선 drawn_humanoid_detector.mar 모델을 TorchServe 엔진에 적재하여
캐릭터의 객체 위치(Bounding Box)와 전반적인 뼈대를 추출한다6, 7. 이후
drawn_humanoid_pose_estimator.mar 모델이 머리, 팔, 다리 등 세부 관절(Joint)의
위치를 정밀하게 예측한다6, 7. 이렇게 추출된 관절 데이터는 2D 캐릭터가 3D
모션을 수행할 수 있는 리깅(Rigging) 기반을 형성한다8.

● MESA 헤드리스(Headless) 렌더링 및 모션 데이터(BVH) 매핑 원리클라우드
서버(Kaggle 등) 환경은 디스플레이 모니터가 존재하지 않는 헤드리스(Headless)
환경이므로, 3D 그래픽 렌더링 시 치명적인 에러가 발생한다9. K-Ride 시스템은
설정 파일(mvc_base_cfg.yaml) 내부의 USE_MESA 옵션을 True로 강제하고, 환경
변수에 PYOPENGL_PLATFORM="osmesa"와 MPLBACKEND="Agg"를 주입하여
가상의 도화지(OSMesa)를 펼치는 방식으로 이를 돌파한다9-11.렌더링 단계에서는
좀비 걸음(zombie.yaml), 춤(jesse_dance.yaml) 등의 모션 캡처 데이터(BVH)가
호출된다12. 3D 모션 데이터의 축(Y축 기준 등)과 거대한 크기 스케일을 캐릭터에
맞게 조정(Scale: 0.005)한 뒤12, PCA 연산과 ARAP(As-Rigid-As-Possible)
알고리즘을 통해 3D 관절의 움직임을 2D 이미지 뼈대에 수학적으로 정렬하고
투영하여 최종적인 애니메이션 프레임(.gif / .mp4)을 그려낸다9, 12, 13.
3.3. AI 한국어 음성 및 배경음악 합성
영상 생성이 완료되면(또는 병렬로), K-Ride 시스템은 영상에 어울리는 한국어 내레이션과
배경음악(BGM)을 합성하여 결합한다.
● GPT-SoVITS V3 (RoBERTa + BigVGAN)를 활용한 고품질 TTS 생성 원리한국어
대본은 최첨단 오픈소스 음성 합성 모델인 GPT-SoVITS (V3)를 통해 고품질
오디오(.wav)로 변환된다14. 텍스트 분석(Text-to-Semantic) 단계에서는
chinese-roberta-wwm-ext-large와 같은 언어 모델(RoBERTa)이 형태소와 문맥을
파악하며, 한국어 처리를 위해 Mecab C++ 엔진(mecab-ko-dic)을 시스템 레벨에
이식하여 세그멘테이션 폴트(Segmentation Fault) 없이 안정적으로 형태소를
분절한다14, 15. 분석된 의미 데이터는 최종적으로 BigVGAN 보코더(Vocoder)를
통해 실제 사람의 목소리와 유사한 자연스러운 음성 파형으로 변환된다14. 특히
BigVGAN 가중치 다운로드 시 발생하는 라이브러리 충돌을 막기 위해 실행 시점에
_from_pretrained 메서드를 강제로 조작하는 몽키 패치(Monkey Patching) 기법을
사용하여 서버 안정성을 극대화하였다16, 17.
● Facebook MusicGen을 활용한 텍스트 프롬프트 기반 맞춤형 BGM 자동 생성영상
분위기를 고조시키는 배경음악은 facebook/musicgen-small 모델에 텍스트
프롬프트를 입력하여 생성한다5, 18. 예를 들어, LLM이 생성한 "A cheerful acoustic
guitar travel vlog instrumental, happy, bright"와 같은 프롬프트를 MusicGen 모델의
파이프라인에 주입하면, 해당 맥락에 맞는 분위기의 BGM 파일(.wav)이 자동으로
합성된다5, 18.
이렇게 독립적으로 생성된 애니메이션(비디오), 한국어 TTS(오디오), BGM(음악) 트랙은
마지막 미디어 믹싱 단계에서 FFmpeg 파이프라인(-filter_complex "[2:a]volume=-18dB")을
통해 BGM의 볼륨을 낮추고, 영상과 오디오의 길이를 자동으로 동기화(-shortest)하여
하나의 완벽한 여행 브이로그 영상(MP4)으로 병합된다5, 19.
위 내용으로 3장의 작성이 완료되었습니다. 이어서 가장 기술적 난이도가 높았던 제4장.
